### Реализованный функционал:

#### ✅ 1. Query Generator Agent
- Генерация 2-3 поисковых запросов из одного вопроса
- Переформулирование и расширение запросов
- Подбор оптимальных весов для гибридного поиска
- Выбор стратегии поиска (concat/separate)
- Определение необходимости уточняющих вопросов

**Пример выхода:**
```json
{
  "clarification_needed": false,
  "queries": [
    {"text": "документы для подключения участка", "reason": "..."},
    {"text": "порядок технологического присоединения", "reason": "..."},
    {"text": "требования к заявителю", "reason": "..."}
  ],
  "search_params": {
    "k": 10,
    "pref_weight": 0.4,
    "hype_weight": 0.3,
    "lexical_weight": 0.2,
    "contextual_weight": 0.1,
    "strategy": "concat"
  },
  "confidence": 0.85,
  "reasoning": "Запрос понятный, используем конкатенацию"
}
```

#### ✅ 2. Search Tool (Гибридный поиск)
- 4 компонента поиска:
  - **pref**: семантический вектор (summary + content)
  - **hype**: семантический вектор (hypothetical questions)
  - **lexical**: BM25 (токенизированный текст)
  - **contextual**: семантический вектор (соседние чанки)
- Поддержка стратегий:
  - `concat`: конкатенация запросов
  - `separate`: раздельный поиск
- Нормализация весов и scores

#### ✅ 3. Search Agent
- Tool Calling для принятия решений
- Автоматическая повторная попытка при плохих результатах
- Оценка качества результатов поиска
- Форматирование результатов для LLM

#### ✅ 4. Response Agent
- Генерация дружелюбных ответов
- Цитирование источников
- Поддержка контекста диалога
- Генерация уточняющих сообщений

#### ✅ 5. Системный промпт
- Дружелюбный стиль общения
- Инструкция по задаванию уточняющих вопросов
- Примеры работы
- Рекомендации по подбору весов

**Пример уточняющего вопроса:**
```
Здравствуйте! Чтобы я мог точно ответить на ваш вопрос 
«Сколько стоит?», мне нужно немного уточнить:

1. Вас интересует стоимость подключения к электросетям?
2. Или вас интересуют тарифы на электроэнергию?
3. Вы физическое или юридическое лицо?

Пожалуйста, ответьте на эти вопросы, и я с радостью помогу вам!
```

#### ✅ 6. Benchmark
- 10 тестовых примеров в `tests/benchmark_samples.json`
- Метрики:
  - Время выполнения (retrieval, generation, total)
  - Уверенность системы
  - Количество запросов на уточнение
  - Среднее количество источников
- Сохранение результатов в CSV и JSON

#### ✅ 7. Тесты
- Юнит тесты для всех агентов
- Тесты для SearchTool
- Интеграционные тесты для AgenticRAG

### Команды для работы:

#### Запуск интерактивно:
```bash
cd agentic_rag
python main.py
```

#### Одноразовый запрос:
```bash
python main.py "Как подать заявку на подключение?"
```

#### С подробным выводом:
```bash
python main.py "Как подать заявку?" --verbose
```

#### Запуск тестов:
```bash
python tests/test_agent.py
# или
pytest tests/test_agent.py -v
```

#### Запуск benchmark:
```bash
python benchmark.py --use-default --output results/
```

### Ключевые особенности:

1. **Автоматический подбор весов**
   - LLM сама решает какие веса использовать
   - В зависимости от типа вопроса

2. **Генерация нескольких запросов**
   - 2-3 запроса из одного вопроса
   - Конкатенация или раздельный поиск

3. **Уточняющие вопросы**
   - При неопределённости задаёт вопросы
   - Дружелюбный стиль общения

4. **Tool Calling**
   - SearchTool как инструмент для LLM
   - LLM решает когда и как искать

5. **Автоматическое исправление**
   - Повторная попытка при плохих результатах
   - Изменение стратегии поиска
