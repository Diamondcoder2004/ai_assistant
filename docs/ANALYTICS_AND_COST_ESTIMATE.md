## 📈 1. Аналитика по вопросам пользователей

### 1.1 Источники данных

Система собирает детальную статистику через:

| Компонент | Хранилище | Данные |
|-----------|-----------|--------|
| **Чаты** | Supabase (таблица `chats`) | Вопрос, ответ, параметры поиска, источники |
| **Фидбек** | Supabase (таблица `feedback`) | Лайки, дизлайки, оценки 1-5, комментарии |
| **Тайминги** | In-memory + файлы логов | Время работы агентов, P95/P99 перцентили |
| **Бенчмарки** | JSON/CSV отчёты | Метрики качества (relevance, completeness, helpfulness) |

### 1.2 Эндпоинты для аналитики

```
GET /history              # История чатов с пагинацией
GET /history/sessions     # Группировка по сессиям и дням
GET /history/{chat_id}    # Детали конкретного чата
GET /feedback/{query_id}  # Фидбек по запросу
GET /timing/stats         # Статистика таймингов
GET /timing/stats/print   # Вывод сводки в лог
POST /timing/stats/save   # Сохранение статистики в файл
```

## 🖥️ 2. Требования к инфраструктуре

### 2.1 Архитектура системы

```
┌─────────────────────────────────────────────────────────┐
│                    Пользователи                         │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  Frontend (Vue.js 3, Nginx)                             │
│  - Порт: 80                                             │
│  - Статика: ~5 MB                                       │
│  - RAM: 50-100 MB                                       │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  Backend (FastAPI, Python 3.11)                         │
│  - Порт: 8880                                           │
│  - Агенты: QueryGenerator, SearchAgent, ResponseAgent   │
│  - RAM: 500 MB - 2 GB (зависит от нагрузки)             │
│  - CPU: 2-4 ядра (для параллельных запросов)            │
└────────────────────┬────────────────────────────────────┘
                     │
         ┌───────────┼───────────┐
         │           │           │
         ▼           ▼           ▼
    ┌─────────┐ ┌─────────┐ ┌──────────────┐
    │ Qdrant  │ │ Supabase│ │ RouterAI API │
    │ (векторы│ │ (БД,    │ │ (LLM,        │
    │ ~2 ГБ)  │ │ auth)   │ │ embedding)   │
    │ RAM:    │ │ External│ │ External     │
    │ 1-2 ГБ  │ │         │ │              │
    └─────────┘ └─────────┘ └──────────────┘
```

| Сервис | Назначение | Трафик | Стоимость |

```yaml

Qdrant: Отдельный контейнер, выделенная память
```

## 📋 7. Чек-лист перед развёртыванием

### Инфраструктура

- [ ] Выбран тариф VPS (рекомендуется Cloud 80 для старта)
- [ ] Настроен IPv4 (публичный IP)
- [ ] Включены ежедневные бэкапы
- [ ] Настроена DDoS-защита (базовая)
- [ ] Открыты порты: 80 (HTTP), 443 (HTTPS), 8880 (API)

### Приложения

- [ ] Docker и Docker Compose установлены
- [ ] Qdrant развёрнут (Docker, порт 6333)
- [ ] Backend настроен (.env: RouterAI, Supabase, Qdrant)
- [ ] Frontend собран (VITE_* переменные)
- [ ] Nginx настроен (reverse proxy для backend)

### Мониторинг

- [ ] Настроено логирование (логи в файлы, ротация)
- [ ] Подключён Uptime-мониторинг (Uptime Robot)
- [ ] Настроены алерты (Telegram, email)
- [ ] Включена статистика таймингов (/timing/stats)

### Безопасность

- [ ] JWT_SECRET_KEY изменён на уникальный
- [ ] SUPABASE_JWT_SECRET настроен
- [ ] HTTPS настроен (Let's Encrypt)
- [ ] Доступ к Supabase ограничен по IP (опционально)
- [ ] Бэкапы тестируются (восстановление)
