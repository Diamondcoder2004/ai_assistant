## 📋 Ключевые возможности

### ✅ Что умеет система

- 🤖 **Автоматический поиск** — LLM сам подбирает параметры поиска
- 💬 **Поддержка контекста** — помнит историю диалога
- 📚 **Источники** — цитирование документов с оценками релевантности
- 📋 **Копирование** — копирование ответов в буфер обмена
- 👍 **Оценка** — лайк/дизлайк/звёзды для улучшения системы
- 📜 **История** — сохранение и возобновление диалогов
- 📊 **Таблицы** — красивое форматирование таблиц
- 🧮 **LaTeX** — поддержка математических формул

### ⚙️ Технические особенности

- **Agentic подход** — LLM агенты принимают решения
- **Гибридный поиск** — 4 компонента (pref, hype, contextual, lexical)
- **Рекомендации** — пользователь может дать подсказки (не жёстко)
- **Streaming** — потоковая передача ответов (SSE)
- **Аутентификация** — JWT токены Supabase

## 📊 Архитектура в двух словах

```
Клиент (Vue 3)
    ↓
FastAPI (Бэкенд)
    ↓
Agentic RAG (Ядро)
    ├─ Query Generator (генерация запросов)
    ├─ Search Agent (поиск)
    └─ Response Agent (ответы)
    ↓
Hybrid Search (Qdrant + BM25)
    ↓
Supabase (история + фидбек)
```

**Подробности:** [ARCHITECTURE.md](ARCHITECTURE.md)

## 🔧 Параметры поиска

Система **автоматически** подбирает параметры:

| Параметр | Описание | LLM решает |
|----------|----------|------------|
| **k** | Количество документов | ✅ Да |
| **pref_weight** | Вес префиксного эмбеддинга | ✅ Да |
| **hype_weight** | Вес hypernym эмбеддинга | ✅ Да |
| **lexical_weight** | Вес BM25 | ✅ Да |
| **contextual_weight** | Вес контекстуального | ✅ Да |
| **strategy** | Стратегия поиска | ✅ Да |

**Рекомендации от пользователя:** Можно указать (`k=20`, `temperature=0.9`), но LLM принимает финальное решение.
