# Документация Agentic RAG системы

## О проекте

**Agentic RAG** — интеллектуальная система поиска и генерации ответов на основе базы знаний компании «Башкирэнерго».

Система использует LLM (Large Language Model) агентов для:
- 🤖 Автоматической генерации поисковых запросов
- ⚖️ Подбора оптимальных параметров поиска
- 📚 Поиска в базе знаний (гибридный поиск)
- ✍️ Генерации ответов с цитированием источников
- 💬 Поддержки контекста диалога

## Как это работает (для пользователя)

### 1. Задайте вопрос

Введите вопрос в поле ввода внизу страницы.

**Примеры вопросов:**
- "Как подать заявку на технологическое присоединение?"
- "Какие документы нужны для подключения земельного участка?"
- "Сколько стоит подключение к электросетям?"

### 2. Получите ответ

Система:
1. **Анализирует вопрос** — определяет тему и намерение
2. **Генерирует поисковые запросы** — 2-3 варианта для лучшего покрытия
3. **Ищет в базе знаний** — находит релевантные документы
4. **Формирует ответ** — с цитированием источников

### 3. Изучите источники

Нажмите **"Показать источники"** чтобы увидеть:
- Названия документов
- Разделы где найдена информация
- Оценки релевантности (смысловая, словесная, общая)

### 4. Оцените ответ

Используйте кнопки под ответом:
- 👍 **Полезно** — ответ помог
- 👎 **Не полезно** — ответ не помог
- ⭐ **Оценить звёздами** — детальная оценка (1-5)
- 📋 **Копировать** — скопировать ответ в буфер обмена

## Возможности системы

### ✅ Что можно делать

| Возможность | Описание |
|-------------|----------|
| **Задавать вопросы** | О технологическом присоединении, тарифах, документах |
| **Вести диалог** | Система помнит контекст предыдущих вопросов |
| **Получать источники** | К каждому ответу прилагаются документы |
| **Копировать ответы** | Кнопка копирования в каждом ответе |
| **Оценивать ответы** | Лайк/дизлайк/звёзды для улучшения системы |
| **Смотреть историю** | Все ваши диалоги сохраняются |
| **Возобновлять диалог** | Вернитесь к предыдущей сессии |

### ❌ Что нельзя делать

| Запрет | Причина |
|--------|---------|
| **Оскорбления** | Система не отвечает на некорректные вопросы |
| **Спам** | Бессмысленные запросы не обрабатываются |
| **Конфиденциальные данные** | Не передавайте персональные данные |
| **Обход ограничений** | Попытки взлома блокируются |

## Технические детали

### Параметры поиска

Система **автоматически** подбирает оптимальные параметры:

| Параметр | Описание | Диапазон |
|----------|----------|----------|
| **k** | Количество документов | 5-15 |
| **pref_weight** | Вес префиксного эмбеддинга | 0.0-1.0 |
| **hype_weight** | Вес hypernym эмбеддинга | 0.0-1.0 |
| **lexical_weight** | Вес BM25 поиска | 0.0-1.0 |
| **contextual_weight** | Вес контекстуального | 0.0-1.0 |
| **strategy** | Стратегия поиска | concat/separate |

**Рекомендации от пользователя:**
Вы можете указать предпочтения (например, `k=20`), но LLM принимает финальное решение на основе анализа вопроса.

### Форматирование ответов

Система поддерживает:
- **Markdown** — заголовки, списки, таблицы
- **LaTeX** — математические формулы `$$E=mc^2$$`
- **Таблицы** — красивые таблицы с чередованием строк
- **Ссылки на источники** — кликабельные `[1]`, `[2]`

## API Reference

### POST /query

**Описание:** Отправка вопроса и получение ответа

**Request:**
```json
{
  "query": "Как подать заявку?",
  "k": 10,              // опционально (рекомендация)
  "temperature": 0.8,   // опционально (рекомендация)
  "max_tokens": 2000,   // опционально (рекомендация)
  "session_id": "uuid"  // опционально (ID сессии)
}
```

**Response:**
```json
{
  "query_id": "uuid",
  "session_id": "uuid",
  "answer": "Текст ответа...",
  "sources": [
    {
      "id": 1,
      "filename": "Правила ТП",
      "breadcrumbs": "Документы → Нормативные",
      "score_hybrid": 0.85,
      "score_semantic": 0.82,
      "score_lexical": 0.78
    }
  ],
  "search_params": {
    "k": 12,
    "pref_weight": 0.4,
    "hype_weight": 0.2,
    "lexical_weight": 0.3,
    "contextual_weight": 0.1,
    "strategy": "concat",
    "queries": ["запрос 1", "запрос 2"]
  },
  "confidence": 0.85
}
```

### POST /query/stream

**Описание:** Потоковый ответ (Server-Sent Events)

**Формат:**
```
data: {"token": "Часть "}
data: {"token": "ответа "}
data: {"done": true, "query_id": "...", "sources": [...]}
```

### GET /history

**Описание:** Получение истории диалогов

**Response:**
```json
[
  {
    "id": 1,
    "session_id": "uuid",
    "question": "Как подать заявку?",
    "answer": "Для подачи...",
    "sources": [...],
    "created_at": "2026-03-23T14:30:00Z"
  }
]
```

### POST /feedback

**Описание:** Отправка обратной связи

**Request:**
```json
{
  "query_id": "uuid",
  "feedback_type": "like",  // like/dislike/star
  "rating": 5,              // только для star
  "comment": "Отличный ответ!"
}
```

## Безопасность

### Аутентификация

- JWT токены от Supabase Auth
- Токен передаётся в заголовке `Authorization: Bearer <token>`
- Срок действия токена проверяется автоматически

### Защита данных

- Персональные данные не сохраняются
- Логи не содержат конфиденциальной информации
- HTTPS для продакшена
