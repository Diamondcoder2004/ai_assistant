ЗАДАНИЕ ПО ИСПОЛЬЗОВАНИЮ ИСКУССТВЕННОГО ИНТЕЛЛЕКТА (ИИ)

При оказании услуг по технологическому присоединению потребителей (ТПП) возникает необходимость взаимодействия с потребителями по вопросам ТП.

Задача подготовить ИИ - консультанта потребителей услуги «Технологическое присоединение к электрическим сетям ООО «Башкирэнерго» энергопринимающих устройств юридических и физических лиц…».
В Приложении 4.1 представлены все нормативные документы, которые должен знать специалист, взаимодействующий с потребителями услуги ТПП. Также в Приложении 4.2 присутствует перечень часто задаваемых вопросов с ответами.
Наши ожидания от ИИ - консультанта по технологическому присоединению потребителей это ответы потребителям на любые вопросы в рамках представленного во вложениях контента.

Было разработано веб-приложение для взаимодействия с системой, модуль искусственного интеллекта, для поиска и генерации вопросов, база данных для хранения необходимой информации о пользователях, истории чатов

# Техническое описание ассистента для ООО «Башкирэнерго»
## 1. Назначение системы
Интеллектуальный чат-бот на архитектуре RAG (Retrieval-Augmented Generation) для автоматизации ответов на вопросы клиентов и сотрудников по технологическому присоединению к электрическим сетям. Ответы формируются строго на основе проверенных корпоративных документов.

## 3. Обработка запроса (пошагово)
Векторизация запроса — вопрос пользователя кодируется той же embedding-моделью.
Семантический поиск — в Qdrant отбираются top-K (~30) наиболее близких фрагментов.
Реранкинг — модель bge-reranker оценивает пары (запрос, фрагмент), отбираются фрагменты с score ≥ порога (по умолчанию 0.2).
Формирование промпта — вопрос + top-3 релевантных фрагмента + история диалога + системная инструкция.
Генерация ответа — LLM формирует ответ с обязательными ссылками на источники в формате [1], [2].
Возврат результата — ответ + метаданные источников + возможность оценки пользователем.

## 4. База знаний
Объём: 22 документа, 4 категории (правовые, инструкции, информационные, паспорта услуг).
Источники: нормативные акты РФ, внутренние регламенты ООО «Башкирэнерго», материалы порталов Россетей и СУЭНКО.
Структура хранения: документы обрабатываются в Markdown с сохранением метаданных (заголовки, категория, источник, версия).
Особенности: поддержка версионности документов, приоритет актуальных редакций.

## 7. Описание моделей ИИ
## Модель inception/mercury-2
Mercury 2 — это чрезвычайно быстрое LLM для рассуждений и первое LLM с диффузией рассуждений (dLLM). Вместо последовательной генерации токенов, Mercury 2 производит и уточняет несколько токенов параллельно, достигая более 1000 токенов в секунду на стандартных GPU. Mercury 2 работает в 5 раз быстрее, чем ведущие оптимизированные по скорости LLM, такие как Claude 4.5 Haiku и GPT 5 Mini, при этом стоимость значительно ниже. Mercury 2 поддерживает настраиваемые уровни рассуждений, контекст на 128K, использование инструментов на родном уровне и вывод JSON, выровненный по схеме. Создан для рабочих процессов программирования, где задержка накапливается, а также для работы в реальном времени с голосом/поиском и агентными циклами. Совместим с OpenAI API. # Perplexity: Embed V1 4B

pplx-embed-v1 -4B — это одна из передовых моделей текстового встраивания Perplexity, созданная для работы с данными в реальном мире и в масштабах сети. pplx-embed-v1 оптимизирована для стандартного плотного поиска текста, а модель с 4 миллиардами параметров максимизирует качество поиска.
