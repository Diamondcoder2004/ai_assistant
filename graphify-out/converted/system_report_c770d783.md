# Назначение системы
Полная схема архитектуры системы представлена ниже


Интеллектуальный ассистент предназначен для автоматизации технической поддержки потребителей по услуге «Технологическое присоединение к электрическим сетям ООО Башкирэнерго». Система отвечает на вопросы клиентов, предоставляет пошаговые инструкции, рассчитывает стоимость подключения.
База знаний формируется из 22 нормативных документов, включая ФЗ-35, Постановление 861, паспорта услуг, инструкции.

# 6. Важные аспекты и ограничения
## 6.1. Безопасность и конфиденциальность
Документы базы знаний обрабатываются через облачный сервис RouterAI, что приемлемо для компании, так как нормативные документы являются публичной информацией. Однако личные данные клиентов тоже могут передаваться, если они введут. Сервис работает по ФЗ 152 о  «О персональных данных»
– Аутентификация через Supabase JWT
– Row Level Security (RLS) в PostgreSQL
– Нормативные документы — публичная информация

## 6.2. Зависимость от внешних сервисов
Система зависит от облачного сервиса RouterAI для LLM-запросов и эмбеддингов. При недоступности RouterAI генерация ответов невозможна.
– RouterAI — внешний провайдер LLM (Mercury 2, pplx-embed)
– При недоступности — fallback на оператора
– Отсутствие кэширования запросов (планируется)
## 6.3. Обновление базы знаний
База знаний требует ручного обновления при изменении нормативных документов. Процесс обновления: загрузка новых документов → парсинг → фрагментация → LLM-обогащение → загрузка в Qdrant.
