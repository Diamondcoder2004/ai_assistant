## 6. Development Roadmap

### Immediate Actions (Week 1-2)

1. **Fix ДУ category (32.8% accuracy)**: The model confuses ДУ procedures with ТП. Need separate prompt for ДУ questions.
2. **Improve search quality for deadlines/costs**: Model frequently gives wrong numbers. Add structured data tables to vector DB.
3. **Fix password recovery procedures**: Model applies generic recovery to all cases, ignoring key context (ЦОК registration, ЮЛ status).

### Short-term (Week 3-4)

1. **Add Anna's corrections to prompts**: Incorporate domain expert feedback into system prompt.
2. **Improve source attribution**: Model says 'not found' 11 times where answer exists. Fix search recall.
3. **Power/category constraint awareness**: Add explicit rules about ФЛ≤15kW, ИП/ЮЛ≤150kW to response agent.

### Medium-term (Month 2-3)

1. **Separate response agents per category**: Different system prompts for ЛК/ДУ/ТПП.
2. **RAG quality monitoring**: Track context_recall scores, alert on degradation.
3. **Benchmark automation**: Run weekly benchmark runs, track accuracy trends.
4. **Expand benchmark**: Add more ДУ-specific and edge case questions.
