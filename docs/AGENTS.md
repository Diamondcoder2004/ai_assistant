# 🤖 Как работают ИИ-агенты в системе

Этот документ объясняет работу трёх агентов, которые обрабатывают ваш вопрос в системе ИИ-ассистента.

---

## 📋 Общая схема работы

```
┌─────────────────────────────────────────────────────────────────┐
│                    Ваш вопрос:                                   │
│         "Как подать заявку на подключение к электросети?"       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  1️⃣ Query Generator Agent                                       │
│     "Понимаю вопрос, генерирую поисковые запросы"               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2️⃣ Search Agent                                                │
│     "Ищу информацию в базе знаний по запросам"                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3️⃣ Response Agent                                              │
│     "Формирую красивый ответ с источниками"                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Ваш ответ:                                    │
│     "Для подачи заявки необходимо... [1][2][3]"                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 1️⃣ Query Generator Agent — Генератор запросов

### 🎯 Что делает

**Простыми словами:** Этот агент — «переводчик» с человеческого языка на язык поисковой системы. Он понимает, что вы спросили, и придумывает, как это лучше искать в базе знаний.

### 📥 Что получает на вход

| Параметр | Пример |
|----------|--------|
| `user_query` | `"Как подать заявку на подключение?"` |
| `history` | `"Пользователь спрашивал про документы"` |
| `category` | `"Физическое лицо"` |
| `user_hints` | `{"k": 10, "temperature": 0.8}` (рекомендации от пользователя) |

### 📤 Что возвращает

```json
{
  "clarification_needed": false,
  "clarification_questions": [],
  "queries": [
    {
      "text": "порядок подачи заявки на технологическое присоединение",
      "reason": "основной запрос"
    },
    {
      "text": "необходимые документы для заявки на подключение",
      "reason": "документы"
    },
    {
      "text": "способы подачи заявки лично через интернет",
      "reason": "способы подачи"
    }
  ],
  "search_params": {
    "k": 10,
    "pref_weight": 0.5,
    "hype_weight": 0.0,
    "lexical_weight": 0.3,
    "contextual_weight": 0.2,
    "strategy": "separate"
  },
  "confidence": 0.85,
  "reasoning": "Пользователь спрашивает про процедуру подачи заявки, нужно найти документы и способы"
}
```

### 🔧 Как работает (на пальцах)

1. **Анализ вопроса:** Агент читает ваш вопрос и определяет ключевые понятия
2. **Генерация вариантов:** Создаёт 2-3 разных формулировки одного и того же вопроса
3. **Подбор параметров:** Решает, сколько результатов искать и какие веса использовать
4. **Проверка на неясность:** Если вопрос слишком общий — задаёт уточняющие вопросы

### 💡 Примеры

#### Пример 1: Конкретный вопрос
```
Вопрос: "Какие документы нужны для подключения дома к электросети?"

Результат:
✅ clarification_needed: false
✅ queries: [
    "документы для технологического присоединения частного дома",
    "перечень документов для заявки на подключение электроэнергии"
  ]
```

#### Пример 2: Неясный вопрос
```
Вопрос: "Хочу свет"

Результат:
⚠️ clarification_needed: true
⚠️ clarification_questions: [
    "Вы хотите подключить новый объект или восстановить электроснабжение?",
    "Речь о частном доме, квартире или коммерческом помещении?"
  ]
```

#### Пример 3: Сложный вопрос
```
Вопрос: "Как увеличить мощность с 15 кВт до 50 кВт для производства?"

Результат:
✅ queries: [
    "увеличение мощности технологического присоединения",
    "документы для увеличения мощности с 15 до 50 кВт",
    "процедура изменения мощности для юридических лиц"
  ]
✅ search_params: {
    "k": 15,  # Больше результатов, так как вопрос сложный
    "strategy": "separate"  # Искать по каждому запросу отдельно
  }
```

### 🎛️ Параметры поиска

| Параметр | Что означает | Пример |
|----------|--------------|--------|
| `k` | Сколько результатов искать | `10` = найти 10 документов |
| `pref_weight` | Вес семантического поиска (смысл) | `0.5` = 50% важности |
| `hype_weight` | Вес гипотетических вопросов | `0.0` = не использовать |
| `lexical_weight` | Вес точного совпадения слов | `0.3` = 30% важности |
| `contextual_weight` | Вес контекстного поиска | `0.2` = 20% важности |
| `strategy` | Как искать | `"separate"` = по каждому запросу отдельно |

## 2️⃣ Search Agent — Агент поиска

### 🎯 Что делает

**Простыми словами:** Этот агент — «библиотекарь». Он берёт запросы от Query Generator и ищет документы в базе знаний (Qdrant). Может искать повторно, если нашёл мало результатов.

### 📥 Что получает на вход

| Параметр | Пример |
|----------|--------|
| `user_query` | `"Как подать заявку на подключение?"` |
| `history` | `"Пользователь спрашивал про документы"` |
| `category` | `"Физическое лицо"` |
| `auto_retry` | `true` (искать повторно, если мало результатов) |
| `max_retries` | `2` (максимум 2 повторные попытки) |
| `user_hints` | `{"k": 10, "weights": {...}}` |

### 📤 Что возвращает

```json
{
  "clarification_needed": false,
  "clarification_questions": [],
  "results": [
    {
      "id": "chunk_123",
      "filename": "Правила технологического присоединения.pdf",
      "breadcrumbs": "Главная → Документы → Правила",
      "category": "Нормативные документы",
      "summary": "Порядок подачи заявки на технологическое присоединение",
      "content": "Заявка подается в письменной форме...",
      "score_hybrid": 0.85,
      "score_semantic": 0.90,
      "score_lexical": 0.80
    }
    // ... ещё результаты
  ],
  "queries_used": [
    "порядок подачи заявки на технологическое присоединение",
    "необходимые документы для заявки на подключение"
  ],
  "search_params": {...},
  "confidence": 0.85,
  "reasoning": "..."
}
```

### 🔧 Как работает (на пальцах)

1. **Запрашивает запросы:** Вызывает Query Generator для получения поисковых запросов
2. **Проверяет уточнения:** Если нужны уточнения — возвращает их пользователю
3. **Ищет в базе:** Выполняет поиск по сгенерированным запросам в Qdrant
4. **Оценивает результаты:** Если найдено мало (< 3) — пробует снова с другими параметрами
5. **Возвращает лучшее:** Отдаёт лучшие найденные документы

### 💡 Примеры

#### Пример 1: Успешный поиск
```
Запрос: "документы для подключения"

Результат:
✅ Найдено: 8 документов
✅ queries_used: ["документы для технологического присоединения"]
✅ confidence: 0.85
```

#### Пример 2: Автоматическая повторная попытка
```
Запрос: "редкий вопрос про трансформаторы"

Попытка 1:
❌ Найдено: 2 документа (мало!)

Попытка 2 (auto_retry):
🔄 Изменены параметры: k=15, другие веса
✅ Найдено: 5 документов
```

#### Пример 3: Требуется уточнение
```
Запрос: "хочу подключить что-то"

Результат:
⚠️ clarification_needed: true
⚠️ clarification_questions: [
    "Что именно вы хотите подключить: дом, гараж, производство?",
    "Какая планируемая мощность подключения?"
  ]
```

### 🔄 Стратегии поиска

| Стратегия | Как работает | Когда используется |
|-----------|--------------|-------------------|
| `concat` | Все запросы объединяются в один | Простые вопросы |
| `separate` | Поиск по каждому запросу отдельно | Сложные, многогранные вопросы |

## 3️⃣ Response Agent — Агент формирования ответов

### 🎯 Что делает

**Простыми словами:** Этот агент — «писатель». Он берёт найденные документы и пишет красивый, понятный ответ с цитированием источников.

### 📥 Что получает на вход

| Параметр | Пример |
|----------|--------|
| `user_query` | `"Как подать заявку на подключение?"` |
| `search_results` | `[...]` (5-10 найденных документов) |
| `history` | `"Пользователь спрашивал про документы"` |
| `temperature` | `0.7` (креативность ответа) |
| `max_tokens` | `2000` (максимальная длина) |

### 📤 Что возвращает

```json
{
  "answer": "**Для подачи заявки...** [1][2]",
  "sources": [
    {
      "id": "chunk_123",
      "filename": "Правила технологического присоединения.pdf",
      "breadcrumbs": "Главная → Документы → Правила",
      "category": "Нормативные документы",
      "summary": "Порядок подачи заявки",
      "score_hybrid": 0.85,
      "score_semantic": 0.90,
      "score_lexical": 0.80
    }
    // ... ещё 4 источника
  ],
  "confidence": 0.8,
  "context_used": "..."
}
```

### 🔧 Как работает ранжирование источников

**ResponseAgent использует умное ранжирование источников** при возврате пользователю:

| Критерий | Вес | Описание |
|----------|-----|----------|
| **Количество цитирований** | 50% | Сколько раз источник упомянут в ответе ([1], [2][3]) |
| **Важность (score_hybrid)** | 50% | Релевантность источника из поиска |

**Формула рейтинга:**
```
Если источник цитируется:
  rating = (citation_count × 0.5) + (score_hybrid × 0.5)
Иначе:
  rating = score_hybrid × 0.3  # Пониженный вес
```

**Пример:**
```
Источник [1]: упомянут 3 раза, score_hybrid=0.85
→ rating = (3 × 0.5) + (0.85 × 0.5) = 1.5 + 0.425 = 1.925

Источник [2]: упомянут 1 раз, score_hybrid=0.90
→ rating = (1 × 0.5) + (0.90 × 0.5) = 0.5 + 0.45 = 0.95

Источник [3]: не упомянут, score_hybrid=0.95
→ rating = 0.95 × 0.3 = 0.285
```

**Результат:** Источники сортируются по рейтингу — **наиболее цитируемые и важные будут первыми**.

### 🔧 Как работает (на пальцах)

1. **Форматирует контекст:** Превращает найденные документы в читаемый текст
2. **Добавляет историю:** Учитывает предыдущие вопросы пользователя
3. **Генерирует ответ:** LLM пишет ответ на основе контекста
4. **Извлекает источники:** Сохраняет информацию о документах для отображения
5. **Проверяет цитаты:** Убеждается, что источники указаны правильно [1][2][3]

### 💡 Примеры

#### Пример 1: Развёрнутый ответ
```
Вопрос: "Как подать заявку?"

Найдено: 5 документов

Ответ:
**Для подачи заявки необходимо:**

1. **Подготовьте документы** [1][2]:
   - Паспорт
   - Документы на собственность
   - Схема расположения энергопринимающих устройств

2. **Выберите способ подачи** [3]:
   - Лично в офисе
   - Через сайт
   - Почтой

3. **Дождитесь подтверждения** [1]
```

#### Пример 2: Ответ с уточнением
```
Вопрос: "Сколько стоит?"

Clarification questions от Query Generator:
["Для какого объекта: дом, квартира, производство?", "Какая мощность нужна?"]

Ответ Response Agent:
"Здравствуйте! Чтобы я мог точно рассчитать стоимость, уточните:
1. Для какого объекта нужно подключение: дом, квартира или производство?
2. Какая планируемая мощность подключения?

После этого я смогу дать точный расчёт стоимости!"
```

#### Пример 3: Недостаточно информации
```
Вопрос: "Редкий вопрос про специфичное оборудование"

Найдено: 0 документов

Ответ:
"К сожалению, в моей базе знаний недостаточно информации по этому вопросу. 
Рекомендую обратиться напрямую в техническую поддержку Башкирэнерго 
по телефону [номер] или через личный кабинет."
```

### 📝 Правила цитирования

Агент обучен правильно цитировать источники:

| Правильно ✅ | Неправильно ❌ |
|-------------|---------------|
| `[1]`, `[2]`, `[3]` | `"источник 1"`, `"источник [1]"` |
| `[1][3]` (несколько) | `"[src_1]"`, `"[документ 1]"` |

## 🎯 Сводная таблица агентов

| Агент | Что делает | Что получает | Что возвращает | Ранжирование |
|-------|-----------|--------------|----------------|--------------|
| **Query Generator** | Генерирует поисковые запросы | Вопрос пользователя | 2-3 запроса + параметры поиска | ❌ Не ранжирует |
| **Search Agent** | Ищет в базе знаний | Запросы от Query Generator | Найденные документы | ✅ По `score_hybrid` (релевантность) |
| **Response Agent** | Пишет ответ | Документы от Search Agent | Текст ответа + источники | ✅ По цитированиям + важности |

### 🔍 Детали ранжирования

| Агент | Критерий | Формула |
|-------|----------|---------|
| **Search Agent** | Релевантность поиска | `pref×0.5 + hype×0.3 + lexical×0.2` |
| **Response Agent** | Важность источника | `цитирования×0.5 + score_hybrid×0.5` |

## 🔧 Технические детали

### Промпты

Каждый агент использует системные промпты:

- **Query Generator:** `prompts/query_generation.py` — инструкция по генерации запросов
- **Response Agent:** `prompts/system_prompt.py` — инструкция по формированию ответов

### LLM Модель

Все агенты используют одну модель через RouterAI:
- Модель: `inception/mercury-2` (настраивается в `config.py`)
- API: OpenAI-совместимый

### Веса поиска

Веса гибридного поиска настраиваются в `.env`:
```bash
RETRIEVE_PREF_WEIGHT=0.5       # Семантический (смысл)
RETRIEVE_HYPE_WEIGHT=0.0       # Гипотетические вопросы
RETRIEVE_LEXICAL_WEIGHT=0.3    # Точное совпадение слов
RETRIEVE_CONTEXTUAL_WEIGHT=0.2 # Контекстный поиск
```
