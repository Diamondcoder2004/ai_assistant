ТЕХНИЧЕСКОЕ ЗАДАНИЕ
РАЗРАБОТКА ИНТЕЛЛЕКТУАЛЬНОГО АССИСТЕНТА НА ОСНОВЕ МЕТОДОВ ОБРАБОТКИ ЕСТЕСТВЕННОГО ЯЗЫКА ДЛЯ АВТОМАТИЗАЦИИ ТЕХНИЧЕСКОЙ ПОДДЕРЖКИ НУЛЕВОГО УРОВНЯ В МОДУЛЕ “1С: УЧЕТ ТЕХНОЛОГИЧЕСКОГО ПРИСОЕДИНЕНИЯ”











г. Уфа, 2026
# 1. Наименование программы

Программное средство «Интеллектуальный помощник потребителя» для автоматизации консультационной поддержки по вопросам технологического присоединения к электрическим сетям.
# 2. Основание для разработки

Разработка осуществляется в рамках выполнения выпускной квалификационной работы для автоматизации обработки обращений потребителей в ООО «Башкирэнерго» (адрес: 450008, Республика Башкортостан, г. Уфа, ул. Российская, д. 56; основной вид деятельности — передача и распределение электроэнергии; численность персонала — свыше 5 000 сотрудников; дата основания — 1995 год).
# 3. Область применения программы

Программное средство применяется в процессе оказания услуги «Технологическое присоединение» и интегрируется с существующими каналами взаимодействия с потребителями:
- веб-портал компании;
- официальный Telegram-канал;
- колл-центр;
- модуль «1С: Учет технологического присоединения».
# 4. Цель разработки

Цель разработки — повышение эффективности консультационной поддержки потребителей услуги технологического присоединения за счёт:
- сокращения среднего времени обработки типового обращения с 8 минут до 2 минут;
- снижения нагрузки на операторов колл-центра на 40–60% путём автоматизации обработки стандартных запросов;
- повышения качества и юридической точности предоставляемой информации через централизованный семантический поиск по нормативным документам;
- обеспечения круглосуточной доступности консультационной поддержки без увеличения численности персонала.
# 5. Целевая аудитория
Физические лица, обращающиеся по вопросам подключения объектов к электросетям;
Юридические лица и индивидуальные предприниматели;
Операторы колл-центра ООО «Башкирэнерго»;
Специалисты отдела технологического присоединения.
# 6. Функциональные требования
## 6.1. Главный интерфейс
Отображение окна для ввода вопроса в свободной форме;
Кнопки быстрого доступа к часто задаваемым темам;
Раздел «Помощь» с инструкцией по использованию;
Возможность переключения на оператора.
## 6.2. Обработка обращения
Приём текста вопроса от пользователя на естественном языке;
Автоматический поиск ответа в базе нормативных документов и часто задаваемых вопросов;
Формирование ответа с указанием источника информации (название документа, пункт, страница);
Оценка уверенности системы в правильности ответа;
При низкой уверенности — предложение уточнить вопрос или подключение оператора.
## 6.3. Работа с базой знаний
Возможность загрузки новых документов в систему без участия программиста;
Автоматическое обновление поискового индекса при изменении нормативной базы;
Ведение журнала обращений и выданных ответов для последующего анализа.
## 6.4. Интеграционные функции
Передача данных о заявке в модуль «1С: Учет технологического присоединения»;
Экспорт статистики обращений в формате для аналитики;
Поддержка работы в защищённом контуре компании без передачи данных во внешние сети.

## 6.5. Общие функции
Сохранение истории диалога в личном кабинете пользователя;
Формирование отчёта по обращению с указанием даты, времени и содержания;
Адаптация интерфейса под разные устройства (компьютер, планшет, смартфон);
Поддержка русского языка как основного, с возможностью расширения на другие языки.
# 7. Требования к интерфейсу
Интерфейс на русском языке;
Единый стиль оформления в соответствии с корпоративными стандартами ООО «Башкирэнерго»;
Чёткое разделение на блоки: ввод вопроса, отображение ответа, дополнительные действия;
Наличие визуальных подсказок и примеров формулировок вопросов;
Адаптивный дизайн для корректного отображения на экранах разного размера.
# 8. Технические требования
Тип приложения: веб-сервис с возможностью интеграции в существующую инфраструктуру;
Язык разработки: поддерживаемый в корпоративной среде заказчика;
Совместимость: операционные системы Windows 10/11, современные веб-браузеры;
Требования к оборудованию: возможность работы на стандартных серверах компании;
Безопасность: хранение и обработка данных только внутри защищённого контура организации.
# 9. Требования к документации
Руководство пользователя с пошаговыми примерами использования;
Инструкция для администратора по установке, настройке и обновлению системы;
Описание алгоритмов формирования ответов и работы с базой знаний;
Отчёт о результатах тестирования с примерами типовых сценариев;
Исходные материалы с комментариями для возможности дальнейшего сопровождения.
