## 1. КРИТИЧЕСКАЯ ОШИБКА: Валидация `session_id`

**Серьёзность:** 🔴 HIGH  
**Вопрос:** Все запросы  
**Описание:** Backend отклоняет запросы с `session_id`, не являющимся валидным UUID.

```
Запрос: {"session_id": "test-1", "query": "..."}
Ответ: {"detail":"Внутренняя ошибка: {'message': 'invalid input syntax for type uuid: \"test-1\"', 'code': '22P02'}"}
```

**Ожидание:** `session_id` должен принимать произвольные строки или быть опциональным без ошибок.  
**Факт:** Сервер ожидает UUID-формат, что ломает сценарии, где frontend генерирует session_id не в UUID-формате.  
**Рекомендация:** Сделать валидацию `session_id` опциональной или генерировать UUID на бэкенде при отсутствии/некорректном значении.
